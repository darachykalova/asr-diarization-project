# Анти-мошеннический голосовой агент — дизайн

**Дата:** 2026-07-09
**Статус:** одобрен пользователем (все 5 секций)

## Цель

Система, которая отвечает на входящие звонки вместо человека, в реальном времени
определяет мошенника, и защищает пользователя: с мошенником разговор завершается
и фиксируется, у легитимного звонящего принимается сообщение («хозяин перезвонит»).
По каждому звонку сохраняется запись, транскрипция, хроника, вердикт и краткая выжимка.

Существующий сервис транскрибации/диаризации не изменяется — он используется
как пост-обработка записей звонков.

## Ключевые решения

| Вопрос | Решение | Обоснование |
|---|---|---|
| Канал звонков | Этап 1 — браузер (микрофон + WebSocket), этап 2 — Telegram-юзербот (TDLib/tgcalls) | Skype закрыт; мошенники реально звонят в Telegram; браузер позволяет отладить ядро без телефонии |
| Цель агента | Защита: вердикт «мошенник» → повесить трубку + записать; иначе → принять сообщение | Главная задача — чтобы человек не разговаривал с мошенником |
| Real-time ASR | Vosk (`vosk-model-small-ru`, замена на большую модель возможна) | Whisper не умеет стриминг; Vosk даёт частичный текст с задержкой 0.2–0.5 с на CPU |
| Детекция мошенника | Правила: YAML-сценарии с фразами-маркерами и весами, порог по сумме очков | Офлайн, объяснимо (видно какие фразы сработали), расширяемо без кода |
| Голос агента | Silero TTS: все заготовки заранее синтезированы в кэш, динамические фразы — на лету | Нулевая задержка на заготовках, гибкость на остальном |
| Выжимка звонка | Локальная LLM через Ollama (после звонка, по Whisper-транскрипции) | Настоящий пересказ; офлайн; задержка не критична — работает после звонка |
| Архитектура | Новый контейнер `call-agent` в том же docker-compose | Переиспользуем PostgreSQL, MinIO, Whisper-пайплайн, админку, авторизацию |

## Архитектура

```
                     ЗВОНОК (реальное время)
┌─────────┐  аудио 16кГц кусками ~200мс   ┌──────────────────────────┐
│ Браузер │ ─────── WebSocket ──────────► │       call-agent          │
│ (микро- │                               │  (новый контейнер)        │
│  фон)   │ ◄────── аудио ответа ──────── │ Vosk → Детектор →        │
└─────────┘                               │ Диалог → Silero TTS      │
                                          └──────────┬───────────────┘
                     ПОСЛЕ ЗВОНКА                    ▼
│ Запись → MinIO → существующий Whisper-пайплайн → транскрипция
│ → Ollama-выжимка → админка (вкладка «Звонки»)
```

Поток звонка:
1. Браузер шлёт звук кусками ~200 мс через WebSocket.
2. Vosk распознаёт на лету (частичный + финальный текст).
3. Финальная фраза → `ScamDetector` (обновляет счёт) и `DialogEngine` (выбирает ответ).
4. Маскировка задержки: если ответ не мгновенный — сразу играется филлер
   («сейчас-сейчас…», «а?», кашель), затем основная фраза.
5. Счёт ≥ порога → вердикт «мошенник» → трубка вниз. 3 минуты разговора без
   признаков мошенничества (таймаут настраивается) → режим «оставьте сообщение».
6. Всё аудио звонка (обе стороны) пишется в WAV; по завершении — MinIO +
   Job существующего пайплайна + Ollama-выжимка.

Бюджет задержки ответа: Vosk ~0.3 с + детектор ~0 + кэшированный TTS ~0 = < 1 с.

## Компоненты call-agent (Python)

| Компонент | Ответственность |
|---|---|
| `CallSession` | Состояние одного звонка (приветствие → разговор → вердикт), таймеры, оркестрация |
| `StreamingASR` | Обёртка Vosk: куски звука → частичный/финальный текст |
| `ScamDetector` | Проверка фраз по YAML-сценариям, накопление счёта, улики |
| `DialogEngine` | Машина состояний: выбор реплики из заготовок по ключевым словам и состоянию |
| `TTSService` | Silero: кэш заготовок (мгновенно) + синтез на лету |
| `CallRecorder` | Запись обеих сторон в WAV → MinIO → создание Job |

### Формат сценария (YAML, не код)

```yaml
# scenarios/fake_bank.yaml
name: Фейковый банк
triggers:
  - phrase: ["служба безопасности", "безопасности банка"]
    weight: 40
  - phrase: ["карта заблокирована", "подозрительная операция"]
    weight: 30
  - phrase: ["продиктуйте код", "код из смс", "назовите код"]
    weight: 60
  - phrase: ["оформлен кредит", "кто-то оформил"]
    weight: 35
threshold: 70
```

Стартовые сценарии: фейковый банк, газовая служба, полиция/следователь.

### Реплики агента (YAML)

```yaml
greeting: ["Алло?", "Да, слушаю"]
fillers: ["Сейчас-сейчас, минуточку…", "А? Повторите", "Ой, подождите, очки найду", "<кашель>"]
keep_talking: ["Ага… и что?", "Ой, а куда нажать-то?"]
take_message: ["Хозяина нет дома, он перезвонит. Что передать?"]
```

## Данные

Новые таблицы в существующей PostgreSQL:

```
calls
├─ id UUID
├─ source            "browser" | telegram-юзернейм/номер
├─ started_at / ended_at / duration_sec
├─ verdict           "scam" | "not_scam" | "undetermined"
├─ scenario          сработавший сценарий или null
├─ confidence        0–100
├─ ended_reason      "detected_scam" | "took_message" | "caller_hung_up"
├─ job_id            → jobs.id (существующий пайплайн)
├─ summary           выжимка от Ollama (nullable, заполняется после)
└─ audio_key         запись в MinIO

call_events          хроника звонка (доказательная база)
├─ call_id → calls.id
├─ at                секунды от начала звонка
├─ speaker           "caller" | "agent"
├─ text              фраза
└─ scam_delta        очки, добавленные фразой (0 для агента)
```

Существующие таблицы не изменяются.

## Админка: вкладка «Звонки»

- **Список:** дата, источник, длительность, вердикт (🔴/🟢/⚪), сценарий.
  Фильтры по вердикту/сценарию/дате — по образцу страницы аудиозаписей.
- **Карточка звонка:** вердикт + уверенность + причина завершения; выжимка
  (крупно, вверху); плеер записи; хроника по фразам с таймкодами и подсветкой
  улик; полная транскрипция — сворачиваемая, доступ пишется в журнал аудита
  (Принцип VI конституции).

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| WebSocket оборвался | `ended_reason="caller_hung_up"`, частичная запись сохраняется и обрабатывается |
| Vosk не распознал (шум/тишина) | Фраза игнорируется; по таймауту филлер «Алло? Вас не слышно» |
| TTS не успел | Кэшированный филлер, основная фраза догоняет |
| Ollama упала | Звонок сохранён, `summary` пуст, перегенерация кнопкой |
| Whisper-пайплайн упал | Real-time данные (хроника, вердикт, запись) уже сохранены независимо |
| Ложное срабатывание детектора | Порог настраивается; в хронике видно какие фразы сработали |

Принцип: real-time слой и пост-обработка независимы; звонок никогда не
пропадает целиком.

## Границы прототипа (этап 1)

- Нет Telegram — микрофон в браузере (этап 2).
- Нет идентификации голоса мошенника по базе эмбеддингов — только текст.
- Детекция только по правилам; LLM-классификатор — задел на будущее.
- Один звонок за раз.

## Тестирование

1. `ScamDetector` — юнит: фразы → счёт → вердикт; тест на каждый YAML-сценарий.
2. `DialogEngine` — машина состояний: приветствие/филлер/hangup.
3. `StreamingASR` — интеграционный: WAV кусками → текст → сравнение с эталоном.
4. `CallSession` e2e — симулированный звонок из файла: вердикт, хроника, MinIO, Job.
5. API «Звонков» — по образцу тестов admin_audio (список, фильтры, карточка, авторизация).

Паттерн проекта: pytest, моки MinIO/Celery, БД на реальной сессии.

## Этап 2 (после прототипа): Telegram

Юзербот на отдельном аккаунте (TDLib + tgcalls): принимает входящий звонок,
аудиопоток ходит в тот же `call-agent` вместо браузерного WebSocket.
Меняется только транспорт; ядро (ASR, детектор, диалог, TTS) не трогается.
Риск: библиотеки неофициальные, могут ломаться при обновлениях Telegram.
