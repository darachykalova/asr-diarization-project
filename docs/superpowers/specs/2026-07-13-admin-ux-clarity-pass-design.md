# Админка: правки для ясности интерфейса — дизайн

**Дата:** 2026-07-13
**Статус:** одобрен пользователем (все 3 пункта)

## Цель

Три точечных UX-правки в существующей админ-панели (`frontend/`), найденные
при разборе реального кода на соответствие принципам юзабилити (Nielsen
Norman Group heuristics, паттерны админ-панелей/таблиц данных Pencil &
Paper — см. источники в конце). Все три — устранение конкретных, уже
обнаруженных проблем, не косметика ради косметики.

## Ключевые решения

| Проблема | Решение | Обоснование |
|---|---|---|
| `AudioListPage`, `CallsListPage`, `AuditLogPage` не загружают данные при открытии — нужно нажать кнопку | Добавить `useEffect(() => { load(); }, [])` в каждую (тот же паттерн, что уже используется в `UsersPage`, `AnalyticsPage`, `CallDetailPage`) | «Видимость состояния системы» — пользователь должен понимать, что происходит, не совершая лишних действий |
| На `AudioListPage` 11 полей фильтра видны одновременно | Всегда видны: поиск по тексту + статус + кнопки «Найти»/«Сбросить». Остальные 8 (ID записи, имя/ID спикера, спикеров от/до, длительность от/до, даты от/до) — под кнопкой «Ещё фильтры ▾» | Прогрессивное раскрытие — не показывать сразу то, что нужно не всем |
| Верхнее меню — 8 пунктов в один ряд без группировки (только у супер-админа) | Разделить на «рабочее» (Загрузить/Аудиозаписи/Звонки/Аналитика — всегда видно) и «админское» (Пользователи/Журнал аудита/Настройки/Симулятор — под выпадающим «Управление ▾») | Смысловая группировка снижает нагрузку на восприятие; у модератора пунктов админ-группы и так нет — для него меню не меняется |

## Компоненты

### 1. Автозагрузка (3 файла)

- **`frontend/src/pages/CallsListPage.tsx`**: импортировать `useEffect`,
  добавить `useEffect(() => { load(); }, []);` сразу после определения
  функции `load`.
- **`frontend/src/pages/AuditLogPage.tsx`**: импортировать `useEffect`,
  добавить `useEffect(() => { fetchLog(1); }, []);` сразу после определения
  `fetchLog`.
- **`frontend/src/pages/AudioListPage.tsx`**: импортировать `useEffect`,
  добавить `useEffect(() => { fetchList(); }, []);` сразу после определения
  `fetchList` (использует уже существующие `page`/`sortBy`/`sortOrder` из
  состояния — на монтировании это дефолтные значения: страница 1, сортировка
  по дате загрузки, убывание).

Подсказки-заглушки («Нажмите «Найти»/«Показать» для загрузки») не убираем —
безвредны, будут просто не видны в обычном потоке (закрыты состоянием
загрузки/данными сразу после монтирования).

### 2. Свёрнутые фильтры (`AudioListPage.tsx`)

- Новое состояние: `const [showMore, setShowMore] = useState(false);`
- Форма разбивается на две части:
  - Всегда видимая строка: поле «Поиск по тексту транскрипции», поле
    «Статус», кнопки «Найти»/«Сбросить», и кнопка-переключатель
    «Ещё фильтры ▾» / «Скрыть фильтры ▲» (текст зависит от `showMore`).
  - Условно рендерящийся блок `{showMore && (...)}` с оставшимися 8 полями
    (ID записи, имя спикера, ID спикера, спикеров от, спикеров до,
    длительность от, длительность до, дата с, дата по) — тот же
    `grid-cols-4`, что и раньше.
- Логика фильтрации (`fetchListWith`, `handleSearch`, `handleReset`) не
  меняется — просто часть полей формы скрыта визуально, значения в
  `filters` продолжают участвовать в запросе как обычно.

### 3. Меню «Управление» (`Nav.tsx`)

- `NAV_ITEMS` разбивается на два массива: `PRIMARY_ITEMS` (Загрузить,
  Аудиозаписи, Звонки, Аналитика) и `MANAGEMENT_ITEMS` (Пользователи,
  Журнал аудита, Настройки, Симулятор) — роли у пунктов не меняются.
- Новое состояние `menuOpen` + `ref` на контейнер выпадающего меню + сброс по
  клику снаружи (`document.addEventListener("mousedown", ...)` в
  `useEffect`, без внешних библиотек — в проекте таких зависимостей ещё нет).
- Кнопка «Управление ▾» рендерится только если после фильтрации по роли
  `management.length > 0` (то есть только для `super_admin` —
  для `moderator` этот список пуст и кнопка не появляется, меню выглядит
  как раньше).
- Кнопка/пункты подсвечиваются активным состоянием так же, как остальные
  пункты меню (`text-white underline` при совпадении текущего пути).
- Клик по пункту в выпадающем меню закрывает меню (`onClick={() =>
  setMenuOpen(false)}`) в дополнение к переходу по ссылке.

## Тестирование

У фронтенда нет автотестов (уже проверено ранее — в `package.json` нет
`test`/`vitest`/`jest`). Проверка — вручную:
1. `npm run dev` (или пересборка `docker compose build frontend`).
2. Открыть «Аудиозаписи», «Звонки», «Журнал аудита» — данные должны
   появиться без нажатия кнопки.
3. На «Аудиозаписях» — свернуть/развернуть «Ещё фильтры», убедиться что
   скрытые поля по-прежнему участвуют в поиске.
4. Под супер-админом — открыть меню «Управление», проверить переход по
   пунктам и подсветку активного состояния; кликнуть вне меню — должно
   закрыться.
5. Под модератором — убедиться, что меню выглядит как раньше (без
   «Управление», у него и так нет доступа к этим пунктам).

## Вне рамок

- Адаптив меню под мобильные экраны (бургер-меню и т.п.) — не просили,
  только группировка для широких экранов.
- Изменение состава/порядка самих полей фильтра — не меняем, только
  видимость.
- Другие страницы (например, `UsersPage`) — уже используют `useEffect`,
  трогать не нужно.
