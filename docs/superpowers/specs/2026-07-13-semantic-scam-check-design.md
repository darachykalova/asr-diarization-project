# Проверка мошенника нейросетью (с заглушками) — дизайн

**Дата:** 2026-07-13
**Статус:** одобрен пользователем (все 4 секции)

## Цель

Быстрый поиск по словам (`call_agent/scam_detector.py`) ловит только точные
фразы из YAML-сценариев — перефразированная разводка проходит мимо. Добавляем
второй, более "умный" уровень проверки: когда поиск по словам **набрал баллы,
но не дотянул до порога срабатывания**, агент в фоне спрашивает локальную
нейросеть (Ollama, уже используется в проекте для пересказа звонков —
[[2026-07-09-anti-scam-agent-design]]), похож ли разговор на мошенничество.
Пока нейросеть думает, звонок не виснет — агент тянет время фразами-заглушками
("Сейчас-сейчас, минуточку…"), которые уже существуют в персоне агента, но
пока нигде не подключены.

Если поиск по словам вообще не набрал баллов (0 очков) — нейросеть не
спрашивается. Это осознанный компромисс: полностью иначе сформулированная
разводка, не задевшая ни одной фразы словаря, всё ещё не будет поймана.

## Ключевые решения

| Вопрос | Решение | Обоснование |
|---|---|---|
| Когда спрашивать нейросеть | Только когда поиск по словам "сомневается" — баллы есть, но порог сценария не пройден | Дёшево и быстро в явных случаях; нейросеть подключается только к пограничным |
| Поведение, пока нейросеть думает | Агент отвечает фразой-заглушкой на каждую следующую реплику звонящего, пока проверка не завершится | Естественно звучит, не выдаёт задержку; заглушки уже написаны в `persona/replies.yaml`, просто не использовались |
| Сколько проверок одновременно | Максимум одна на звонок — новое подозрение, пока проверка идёт, не запускает вторую | Проще состояние, не перегружаем Ollama |
| Если нейросеть говорит "мошенник" | Одна проверочная (на самом деле — прощальная) фраза, и сразу трубка | Не настораживает мошенника резкой тишиной/обвинением; трубка кладётся автоматически, не дожидаясь ответа |
| Если нейросеть говорит "не мошенник" | Возврат к обычному разговору, как будто ничего не было | Не усложняем — простое решение по умолчанию |
| Сбой/таймаут нейросети (~15 сек) | Считается как "не мошенник", разговор продолжается | Fail-safe: никогда не виснем и не обвиняем из-за технической ошибки; тот же принцип, что и в `summary.py` ("никогда не бросает исключение") |
| Новые фразы | 2 новые заглушки + новая категория "перед трубкой" (её раньше не было вовсе — трубка сейчас кладётся молча) | Нужны бытовые, не настораживающие поводы закончить разговор |

## Архитектура и поток данных

```
Реплика звонящего
        │
        ▼
Быстрый поиск по словам (мгновенно, как сейчас)
        │
   ┌────┴─────────────────────┐
   │                           │
явно НЕ мошенник         явно МОШЕННИК (порог пройден)
   │                           │
обычный ответ              трубка (как сейчас, без изменений)
                                
БАЛЛЫ ЕСТЬ, НО ПОРОГ НЕ ПРОЙДЕН:
   проверка уже идёт в фоне  →  заглушка
   проверки ещё нет          →  запускаем нейросеть в фоне, заглушка

Каждый следующий ход, пока фоновая проверка не завершилась → заглушка.
Проверка завершилась:
  "не мошенник" → обычный разговор дальше
  "мошенник"    → фраза "перед трубкой" + трубка (одним ходом)
```

Пример:
1. "Ваша карта заблокирована" → баллы есть, порога нет → нейросеть спрошена в
   фоне → агент: *"Сейчас-сейчас, минуточку…"*
2. "Скажите код из смс" → проверка ещё не готова → агент: *"А? Повторите,
   пожалуйста."*
3. Нейросеть отвечает "да" → агент: *"Ой, у меня чайник закипел, я
   перезвоню"* → трубка.

## Компоненты

- **`call_agent/persona/replies.yaml`** — добавить в `fillers`:
  `"Ой, погодите, ручку возьму, чтобы записать."`,
  `"Так, так… минуточку, соображу."`
  Новая категория `before_hangup`:
  `"Ой, у меня чайник закипел, я перезвоню."`,
  `"Кажется, в дверь звонят, извините, мне пора."`,
  `"Ой, второй телефон звонит, мне надо ответить."`

- **`call_agent/scam_detector.py`** — новый метод на `ScamDetector`, дающий
  текущий лучший счёт даже если порог не пройден (сейчас `verdict()` отвечает
  только "scam"/"undetermined", без промежуточного состояния), чтобы
  `session.py` мог определить "сомневается" = `0 < score < threshold`.

- **`call_agent/semantic_check.py`** (новый файл) — функция, которая
  спрашивает Ollama "похож ли разговор на мошенничество?" и возвращает
  bool. Сделана по образцу `call_agent/summary.py` (тот же приём: локальный
  Ollama, инъекция `http_post` для тестов, никогда не бросает исключение —
  при сбое/таймауте возвращает "не мошенник").

- **`call_agent/dialog_engine.py`** — новый метод `before_hangup_line()`
  (по образцу существующего `take_message_line()`), достаёт случайную фразу
  из новой категории `before_hangup`.

- **`call_agent/session.py`** — основные изменения в `CallSession`:
  - новое состояние: запущена ли фоновая проверка и не завершилась ли она;
  - копится текст разговора для контекста, который передаётся нейросети;
  - в `on_pcm()`: если поиск по словам "сомневается" и проверки ещё нет —
    запустить фоновую проверку, ответить заглушкой; если проверка уже идёт —
    снова заглушка; если проверка завершилась — либо обычный ответ, либо
    фраза "перед трубкой" + трубка одним ходом (как уже умеет код в
    `main.py`, отправляющий несколько действий подряд).
  - Фоновая проверка запускается через переданный в конструктор инструмент
    (внедряемая зависимость — как уже сделано с `now=time.monotonic` и
    `recognizer` в `StreamingASR`), чтобы `CallSession` оставался
    тестируемым без реальных потоков и без реальной Ollama.

- **`call_agent/main.py`** — при создании `CallSession` передаётся
  инструмент для запуска фоновой проверки (обёртка над
  `semantic_check.check_scam_semantically`, привязанная к текущим
  `settings`).

## Обработка ошибок

- Сбой или таймаут Ollama (лимит ~15 сек на попытку) → `check_scam_semantically`
  возвращает `False` ("не мошенник"), разговор продолжается штатно.
- Максимум одна фоновая проверка на звонок одновременно — повторное
  "сомнение", пока проверка идёт, не запускает вторую, просто ещё одна
  заглушка.
- Если звонок завершается (звонящий положил трубку сам), пока проверка ещё
  не завершилась — проверка просто доигрывает в фоне и её результат
  отбрасывается, ничего не падает.

## Тестирование

- `scam_detector.py`: новый метод даёт баллы ниже порога корректно (не
  требует прохождения порога, в отличие от `verdict()`).
- `semantic_check.py`: с подставным `http_post` — правильный разбор
  "да"/"нет" из ответа, и `False` при сбое/таймауте (по образцу
  `tests/call_agent/test_summary.py`).
- `dialog_engine.py`: `before_hangup_line()` достаёт фразу из новой
  категории.
- `session.py`: основная логика состояний — заглушка пока проверка идёт,
  обычный разговор либо трубка после завершения проверки — с подставным
  (не настоящим) инструментом фоновой проверки, управляемым вручную в
  тесте (без реальных потоков, без реальной Ollama).
- Ручная проверка: тестовый звонок через браузерный симулятор с фразой,
  которая набирает баллы, но не проходит порог — убедиться, что заглушки
  звучат, и что итоговое решение (разговор дальше / трубка) срабатывает.

## Вне рамок

- Проверка нейросетью реплик с нулевым счётом по словарю — не делаем (см.
  «Цель»); ловим только пограничные случаи.
- Настройка таймаута/модели через отдельные env-переменные — переиспользуем
  существующие `ollama_url`/`ollama_model` из `call_agent/config.py`, таймаут
  зашит в коде как разумное значение.
- Новые сценарии мошенничества (доп. фразы, новые YAML-файлы) — отдельная
  задача после этой, не в этом дизайне.
