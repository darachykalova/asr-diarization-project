# Котик-маскот в наушниках — дизайн

**Дата:** 2026-07-13
**Статус:** одобрен пользователем (внешний вид утверждён через визуальный компаньон, поведение — текстом)

## Цель

Декоративный котик в наушниках сидит в правом нижнем углу на каждой странице
фронтенда (включая страницу входа) и покачивает головой в такт воображаемой
музыке. Чисто украшение — не несёт функциональной нагрузки, ни на что не влияет
и ни на что не реагирует.

## Внешний вид (утверждено через визуальный компаньон)

SVG-котик, мультяшный милый стиль:
- Два коротких треугольных уха с розовой внутренней частью (не длинные, не
  острые — кошачьи пропорции).
- Наушники: единая дужка-градиент (фиолетовые тона) поверх головы + объёмные
  подушечки с бликом (радиальный градиент, светлее в центре).
- Розовый бантик на правом ухе, наклонён по диагонали (~25°), сидит по центру
  уха.
- Плавающая нотка (♪) рядом для настроения.
- Финальная версия зафиксирована в
  `.superpowers/brainstorm/1757-1783943650/content/bobbing-nod-v10.html` —
  именно эту разметку SVG переносим в React-компонент один в один (цвета,
  координаты, градиенты).

## Анимация

Только вертикальный кивок в такт (без наклона/раскачивания влево-вправо):

```css
@keyframes cat-nod {
  0%, 100% { transform: translateY(0); }
  25%      { transform: translateY(7px); }
  50%      { transform: translateY(-1px); }
  75%      { transform: translateY(4px); }
}
```

`transform-origin` — низ фигуры (`50% 92%`), длительность 0.8s,
`cubic-bezier(.45,.05,.55,.95)`, бесконечный повтор. Утверждено пользователем
как финальный вариант («Кивок» без раскачивания).

## Ключевые решения

| Вопрос | Решение | Обоснование |
|---|---|---|
| Где рисуется | Отдельный React-компонент `MascotCat`, рендерится один раз на верхнем уровне `App.tsx` (рядом с `<Routes>`, а не внутри защищённого layout'а) | Страница входа (`/login`) объявлена вне `ProtectedRoute` — чтобы котик был виден и там, компонент не может жить внутри защищённого блока |
| Позиция | `fixed`, правый нижний угол | Как и просил пользователь |
| Конфликт с уведомлениями | `Notifications.tsx` уже занимает правый нижний угол (`fixed bottom-4 right-4`) — поднимаем её отступ снизу, чтобы тосты не перекрывали котика | Оба элемента фиксированы в одном углу; сдвигаем один из них, не трогая внутреннюю логику стека тостов |
| Анимация в коде | Кастомный keyframe `cat-nod` через `tailwind.config.js` (`theme.extend.keyframes/animation`), а не инлайн-стили | Проект уже полностью на Tailwind-классах — не заводим отдельный CSS-файл ради одной анимации |
| Клик | Ничего не делает | Осознанный выбор — чисто декоративный элемент |
| Тесты | Автотестов нет — у фронтенда нет тестового фреймворка вообще (проверено: в `package.json` нет `test`/`vitest`/`jest`) | Проверка — вручную, через dev-сервер |

## Компоненты

- **`frontend/tailwind.config.js`** — добавить в `theme.extend`:
  `keyframes: { catNod: {...} }`, `animation: { "cat-nod": "catNod 0.8s cubic-bezier(.45,.05,.55,.95) infinite" }`.
- **`frontend/src/components/MascotCat.tsx`** (новый) — функциональный
  компонент без пропсов, рендерит `<div className="fixed bottom-4 right-4 z-40">`
  с SVG-котиком внутри (разметка — из утверждённого макета), класс
  `animate-cat-nod origin-[50%_92%]` на обёртке SVG.
- **`frontend/src/components/Notifications.tsx`** — сменить
  `fixed bottom-4 right-4` на `fixed bottom-28 right-4` (строка с классами
  контейнера тостов), чтобы всплывающие уведомления не перекрывали котика.
  Внутренняя логика стека тостов (`flex-col gap-2 items-end`) не меняется.
- **`frontend/src/App.tsx`** — импортировать `MascotCat` и отрендерить
  `<MascotCat />` один раз, сразу после `</Routes>`, внутри `<BrowserRouter>`
  (то есть вне `<Routes>` — рендерится независимо от того, какой роут
  совпал, включая `/login`).

## Проверка

Ручная, через dev-сервер фронтенда:
1. `cd frontend && npm install && npm run dev` (или прод-сборка через
   `docker compose build frontend && docker compose up -d frontend`, если
   нужно посмотреть в уже поднятом стеке — см. раздел «Как посмотреть, что
   всё работает» ниже).
2. Открыть `/login` — котик виден, кивает.
3. Войти и открыть любую страницу админки (например `/audio`) — котик виден
   там же, в том же углу.
4. Спровоцировать всплывающее уведомление (дождаться завершения любой
   задачи/звонка) — убедиться, что тост появляется выше котика, не
   перекрывая его.

## Как посмотреть, что всё работает (для пользователя)

После того как код будет реализован:
- **Через dev-сервер** (быстрее всего для проверки): `cd frontend && npm run dev`,
  открыть `http://localhost:5173` в браузере.
- **Через уже поднятый прод-стек**: `docker compose build frontend && docker compose up -d frontend`,
  открыть `http://localhost:4000` (или порт из `FRONTEND_PORT` в `.env`).

## Вне рамок

- Реакция котика на события (уведомления, ошибки, звонки) — не делаем,
  чисто декоративный элемент.
- Настройка/отключение котика через админ-настройки — не требуется.
- Общий рефакторинг позиционирования `Notifications` (например, общий
  flex-контейнер для обоих элементов) — минимальное точечное изменение
  отступа достаточно для этой задачи.
