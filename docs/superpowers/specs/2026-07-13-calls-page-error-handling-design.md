# Обработка ошибок на странице «Звонки» — дизайн

**Дата:** 2026-07-13
**Статус:** одобрен пользователем

## Цель

`CallsListPage.tsx` — единственная страница-список в админке без обработки
ошибок сети/сервера: если запрос `fetch` падает или сервер отвечает не-2xx,
страница молча ничего не показывает. Соседние страницы (`AuditLogPage.tsx`,
`AudioListPage.tsx`) уже решают эту задачу одинаковым паттерном — переносим
его один в один, без изобретения нового подхода.

## Решение

Скопировать уже существующий и проверенный паттерн из `AuditLogPage.tsx`:

- Новое состояние `const [error, setError] = useState<string | null>(null);`
- `load()` оборачивается в `try/catch/finally`: `setError(null)` в начале,
  `if (!r.ok) throw new Error(\`HTTP ${r.status}\`);` после fetch, `catch (e) {
  setError(String(e)); }`, `setLoading(false)` переносится в `finally`.
- Перед блоком `{data && (...)}` добавляется:
  ```tsx
  {error && <div className="bg-red-50 border border-red-200 text-red-700 rounded p-3 mb-4 text-sm">{error}</div>}
  ```
  (тот же класс, что и в `AuditLogPage.tsx`/`AudioListPage.tsx` — визуальная
  консистентность).

## Компоненты

- **`frontend/src/pages/CallsListPage.tsx`** — единственный файл.

## Тестирование

Автотестов нет. Ручная проверка: временно указать неверный `API_BASE` или
остановить `api`-контейнер, открыть «Звонки» — должна появиться красная
плашка с текстом ошибки вместо молчаливой пустой страницы. Затем вернуть всё
как было и убедиться, что обычная загрузка по-прежнему работает.

## Вне рамок

- Другие страницы уже имеют обработку ошибок — не трогаем.
- Автоматический повтор запроса при ошибке — не просили.
