# Использование навыков emilkowalski/skills — рабочий план

> **Тип документа:** план использования инструментов, не инженерный implementation-план — здесь нет кода и тестов, поэтому обычный формат superpowers:writing-plans (Task/Files/Steps с TDD) не применяется. Это чек-лист по этапам.

**Цель:** встроить 6 навыков про UI-полировку и анимации (установлены глобально из `emilkowalski/skills` 2026-07-16) в работу над `frontend/` этого проекта — сначала разведка, потом реализация, потом проверка.

**Навыки:**
- `animation-vocabulary` — как называется тот или иной эффект (для точных формулировок запроса)
- `apple-design` — референс по «физичным» жестам/переходам в стиле Apple
- `emil-design-eng` — общая философия полировки интерфейса (фон, не отдельный шаг)
- `find-animation-opportunities` — точечно ищет места без анимации, которые её просят (read-only)
- `improve-animations` — широкий аудит + приоритизированный roadmap (read-only)
- `review-animations` — строгая проверка уже сделанной анимации перед мержем

## Этап 1 — разведка (read-only, ничего не меняет в коде)

Кандидаты во `frontend/src/`, с которых стоит начать (недавно дорабатывались в UX-раунде 2026-07-13):
- `components/ConfirmDialog.tsx`
- `components/LoadingSpinner.tsx`
- `components/Notifications.tsx`
- `pages/CallsListPage.tsx`, `pages/AudioListPage.tsx` — списки, есть что оживить при загрузке/удалении строки

Пример запроса: «Посмотри ConfirmDialog.tsx и LoadingSpinner.tsx — что стоит оживить?» → сработает `find-animation-opportunities`.

Если нужен взгляд сразу на всю админку целиком, а не по конкретным файлам — просить `improve-animations` («сделай аудит анимаций по всему frontend/src и дай приоритизированный план»).

Результат этапа: список конкретных предложений (тайминги, кривые), без изменений в коде.

- [x] Сделано 2026-07-16 — 5 находок (ConfirmDialog вход, тосты, аккордеон фильтров, fade строки при удалении, press-feedback кнопок)

## Этап 2 — реализация

Взять находки из этапа 1 и явно попросить реализовать топ-находки. По ходу:
- звать `apple-design` за референсом для конкретного жеста — кандидат: `pages/CallSimulatorPage.tsx` (живой звонок через WebSocket, важны момент и прерываемость перехода) или полупрозрачность/глубина в модалках
- звать `animation-vocabulary`, если не хватает точного термина для формулировки запроса

- [x] Сделано 2026-07-16 — все 5 находок реализованы в `ConfirmDialog.tsx`, `Notifications.tsx`, `AudioListPage.tsx`, `tailwind.config.js`; визуально проверено в браузере (модалка, аккордеон)
- [x] Доп. `apple-design` для `pages/CallSimulatorPage.tsx` (2026-07-16) — кнопка «Позвонить/Завершить» слита в один элемент с color-crossfade вместо подмены, пульсирующий индикатор «В звонке», fade-in строк лога. Визуально проверено живым звонком через WS.

## Этап 3 — проверка перед мержем

Прогонять `review-animations` на diff после того, как анимация сделана, перед коммитом. Навык строгий по умолчанию — находки это норма, не сигнал, что всё плохо.

- [x] Сделано 2026-07-16 — вердикт Block (keyframes на тостах), все находки исправлены: тосты переведены на transitions, везде единая сильная кривая `cubic-bezier(0.23,1,0.32,1)`, reduced-motion сделан «мягче» вместо «ноль»
