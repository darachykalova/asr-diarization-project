## 1. Исходное состояние системы

На момент начала работы проект существовал в виде кода, однако при запуске и тестировании были обнаружены критические проблемы:

| Проблема | Влияние |
|----------|---------|
| `pyannote.audio` отсутствовал в `requirements.txt` | Диаризация падала при каждой задаче, статус всегда `partial` |
| Слова с таймкодами (FR-6) не сохранялись в БД | ASR вычислял метки, но они терялись до записи |
| Флаг перекрытия речи (FR-10) всегда `false` | Overlap не определялся |
| Массив `speakers[]` отсутствовал в ответе | Нарушение формата ТЗ |
| `/healthz` и `/readyz` возвращали 404 | Роутер был прикреплён под `/v1`, а не в корне |
| `GET /v1/jobs/{unknown}` → 200 вместо 404 | Celery возвращает PENDING для любого неизвестного ID |
| `POST /v1/transcriptions` → 200 вместо 202 | Нарушение ТЗ (асинхронный приём = 202) |
| Воркер падал при обработке второй задачи подряд | Docker ограничен 3 ГБ RAM, pyannote требует ~2.5 ГБ |
| Задачи зависали на диаризации 10+ минут | pyannote обращался к huggingface.co при каждом запуске |
| Модели перезагружались с диска при каждой задаче | Главный источник задержек при последовательной обработке |

### 2.2 Офлайн-режим и руководство пользователя (23 июня 2026)

**Коммит `2937b1a`**

#### Устранение зависания диаризации
- **Причина:** pyannote при каждом запуске делал HEAD-запросы к `huggingface.co` для проверки актуальности кэша. При нестабильной сети — зависание на 10+ минут.
- **Решение:** добавлены переменные среды в `docker-compose.yml`:
  ```yaml
  HF_HUB_OFFLINE: "1"
  TRANSFORMERS_OFFLINE: "1"
  ```
- **Результат:** диаризация 4-минутного файла — 10–15 минут вместо зависания.

#### Руководство пользователя (`РУКОВОДСТВО.md`)
- Инструкция по запуску системы
- Получение admin API-ключа
- Доступ к Swagger (самоподписанный сертификат)
- Чек-лист проверки всех требований ТЗ (FR-1 — FR-16, NFR)
- Честный список нереализованного (FR-8)

- **Проблема:** Docker Desktop ограничен 2.83 ГБ (дефолт WSL2). Воркер падал при обработке второй задачи подряд — OOM-килл.
- **Решение:** создан `C:\Users\256bit.by\.wslconfig`:


#### Кэш моделей на уровне процесса (`services/model_cache.py`)
- **Проблема:** модели создавались заново при каждой задаче.

- `--pool=prefork --concurrency=2 --max-tasks-per-child=10`
- `OMP_NUM_THREADS=4`, `MKL_NUM_THREADS=4`, `mem_limit=5g`
- 2 воркера — реальный потолок: каждый с pyannote потребляет ~2.5 ГБ

### 2.6 Разбиение пайплайна на Celery-таски с chain (25 июня 2026)

**Коммит `7809ecf`**

#### Мотивация
Монолитный `process_audio_task` выполнял все шаги последовательно в одной функции. Это затрудняло отдельное масштабирование шагов, ретраи на конкретном шаге и видимость прогресса.

#### Архитектура chain

```
normalize → asr → diarize → merge_align → persist → identify_speakers → finalize
```

Каждый таск принимает и возвращает `dict` (context), обогащая его:

| Таск | Добавляет в контекст |
|------|----------------------|
| `pipeline.normalize` | `normalized_path` |
| `pipeline.asr` | `asr_segments`, `detected_language`, `duration_sec` |
| `pipeline.diarize` | `speaker_segments`, `diarization_error` |
| `pipeline.merge_align` | `aligned_segments`, `full_text` |
| `pipeline.persist` | `transcript_id`, `final_status` |
| `pipeline.identify_speakers` | (пишет в БД и Qdrant, ключей не добавляет) |
| `pipeline.finalize` | (обновляет статус джоба, отправляет webhook) |

#### Переключение режимов

Переменная среды `PIPELINE_MODE`:
- `monolith` (по умолчанию) — старый монолитный таск, безопасный откат
- `chain` — новый разбитый пайплайн

Оба эндпоинта (`/upload` и `/url`) автоматически выбирают режим.

#### Результаты тестирования chain

```
chain job e096b532 — 15 сек аудио, max_speakers=1
[10s] processing progress=20
[30s] processing progress=90
[40s] done progress=100
→ 7 сегментов сохранено в БД, язык: ru
```

### 2.7 Пересборка Docker — модели из локального архива (25 июня 2026)

**Коммит `7809ecf`**

#### Проблема

При `docker compose build` скрипт `download_models.py` скачивал модели с HuggingFace (~650 МБ). Если HuggingFace недоступен → сборка падает. Требовался `HF_TOKEN` для gated-модели pyannote.

#### Решение

Dockerfile больше не скачивает ничего при сборке. Модели берутся при старте контейнера из трёх возможных источников по приоритету:

```
1. models_cache volume уже содержит модели  →  пропуск (мс)
2. ./models_backup/models.tgz               →  распаковка (~30 сек)
3. MinIO bucket ml-models                   →  скачивание по файлам
4. Ничего из вышеперечисленного             →  exit 1, контейнер не стартует
```

#### Результат

| Параметр | До | После |
|----------|:---:|:---:|
| Требует HF_TOKEN | да | нет |
| Требует интернет при сборке | да | нет |
| Размер Docker-контекста | 650 МБ | 140 КБ |
| Время сборки | 10–15 мин | секунды (кэш) |
| Работает без интернета | нет | **да** |

Архив `models_backup/models.tgz` (650 МБ) монтируется в оба контейнера как read-only и не попадает в git-репозиторий.

## 4. Архитектура системы

```
HTTPS (443) ──► nginx ──► FastAPI (8000)
                               │
                    ┌──────────┴──────────┐
               REST API            Celery chain/monolith
                                          │
                                     Redis (очередь)
                                          │
                          ┌───────────────┼───────────────┐
                          │               │               │
                     PostgreSQL         MinIO           Qdrant
                     (метаданные)      (аудио +       (текстовые +
                                      ML-модели)    голосовые векторы)
```

**Pipeline chain (PIPELINE_MODE=chain):**
```
normalize → asr → diarize → merge_align → persist → identify_speakers → finalize
```

**Воркер:** 2 параллельных prefork-процесса, каждый кэширует модели в памяти:
- faster-whisper base (ASR, word timestamps)
- pyannote/speaker-diarization-3.1 (диаризация)
- SpeechBrain ECAPA-TDNN (голосовые embeddings, 192-dim)
- sentence-transformers paraphrase-multilingual-MiniLM-L12-v2 (текстовые embeddings)

## 6. Что не реализовано

| Требование | Причина / план |
|-----------|----------------|
| **FR-8 Стерео-телефония** | Единственное незакрытое обязательное требование. «Канал = спикер» для стерео-звонков. Реализуется отдельной веткой пайплайна по признаку `channels=2` в заголовке аудио. |
| Swagger офлайн | `/docs` загружает UI с `cdn.jsdelivr.net`. При отсутствии интернета — пустая страница. На работу API не влияет. |
| GPU-образ | Требует CUDA. На CPU: 3-минутный файл = ~14 мин. С GPU (RTX 3060): ожидается ~30–40 сек. |
| Замеры WER/DER/EER | Требуют размеченного тест-сета от заказчика. |
