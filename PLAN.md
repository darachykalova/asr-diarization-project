## 4. Нарезка длинного аудио на сегменты

**Решение:** длина куска `CHUNK_DURATION_SEC=300` (5 мин), **configurable** через env.
5 мин — баланс параллельности, надёжной диаризации и памяти (см. ниже). Точное число
подберём эмпирически.

**Идея:** длинное аудио (до 4 ч по ТЗ) резать на куски и обрабатывать покусочно.
Три выигрыша:
1. **Память** — воркер держит в RAM только текущий кусок (решает OOM на длинных файлах).
2. **Параллельность** — куски расходятся по воркерам/слотам, час аудио ≈ время одного куска.
3. **Квалити по кускам** — на каждый кусок своя метрика SNR и своя модель.

**Важно:** нарезка ускоряет только до числа параллельных слотов (`concurrency=2` →
2 куска зараз). Рычаг скорости — число воркеров, нарезка лишь даёт что параллелить.
Крупные куски с `large-v2` параллельно → риск OOM; умеренные 5 мин безопаснее.

**Правила нарезки (краевые случаи):**
1. Файл ≤ ~6 мин (кусок + порог) → **не режем**, обрабатываем целиком.
2. Хвост меньше порога (~2 мин) → **приклеиваем к предыдущему куску**
   (10:30 → 5 + 5:30, а не 5 + 5 + 0:30). 8 мин → 5 + 3 (оба ок).
3. Резать **по тишине** в окрестности цели (±~20 с), а не ровно по таймеру — иначе рвём слово.

**Главная сложность — сшивка кусков:**
- **Спикеры:** локальные метки `SPEAKER_00` в разных кусках — могут быть разными
  людьми. Сводить через голосовые эмбеддинги (механизм есть, подключить к склейке).
- **Таймкоды:** к словам куска N прибавлять смещение (N × длина куска).
- **VAD для резки по тишине:** `vad_service.py` сейчас заглушка (возвращает весь файл
  одним сегментом) — нужен реальный Silero VAD.

**Объём:** несколько часов + тесты. Стандартная техника (так делает WhisperX).
